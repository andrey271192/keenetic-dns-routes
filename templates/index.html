Учётные данные Keenetic (API)

Два варианта по умолчанию: KEENETIC_LOGIN / KEENETIC_PASSWORD в .env на сервере или свои логин/пароль (или http(s)://логин:пароль@хост:порт) у каждого роутера. Пароль входа на эту страницу — только ADMIN_PASSWORD.

В URL можно указать логин:пароль@ перед хостом — при сохранении они переносятся в поля, в адресе остаётся только http(s)://хост:порт. Пути вида /rci/... в base URL не указывай — их добавляет сервис.

US — домены и IP/CIDR (одна строка = одна запись)

RU — домены и IP/CIDR

Роутеры (KeenDNS → base URL прокси)

У каждого роутера: base URL + при необходимости логин/пароль, либо сразу URL http(s)://логин:пароль@хост:порт. Если поля логина/пароля пустые — берутся KEENETIC_* из .env на сервере.

Укажи тот же хост и порт, что в KeenDNS для HTTP Proxy (часто не стандартный порт). Для http без порта запрос уйдёт на 80 и может быть /auth 404. /auth 403 — проверь разрешённые IP для API на роутере.
